# Theo Client Flow Engine Skill

## Purpose

Build and operate the first client-flow system for Howling Mune.

## Inputs Required

- Theo source files in `source/`
- approved strategic notes
- intake schema
- routing schema
- CRM schema
- approved commercial terms
- Airtable, Notion, Stripe, and notification environment values when wiring live integrations

## Source Materials

- Theo Vision Call - 2026-04-21
- Theo Foundational Blueprint
- Theo Blueprint Luxury
- Theo Strategic Handoff
- Theo Creator-Healer OS Setup
- The Creative Container
- Client Flow & Payment Engine notes
- Existing Theo Clarity Blueprint / Notion intake answers if available
- Existing proposal/SOW language if approved

## Outputs

- front door specification
- intake/application form
- route classification
- Airtable lead record
- close sequence support
- Stripe milestone path
- launch checklist
- data review notes

## Rules

- Do not overbuild.
- Do not invent client facts.
- Do not confuse Sheetal language with Theo language.
- Do not finalize pricing automatically.
- Do not create public-facing claims without human review.
- Preserve source documents before summarizing them.
- Use markdown docs first before live code.
- Keep V1 narrow enough to launch.

## First-Build Logic

If the client has real work and proof but no clean pathway, build:

```text
front door -> intake -> routing -> CRM -> close -> payment -> build activation
```

Do not build the full ecosystem until lead data and revenue validate the first path.

## Routing Logic

Supported routes:

- healing_only
- photography_only
- hybrid_identity_reset
- group_or_workshop
- nurture
- partner_or_retreat
- manual_review

Use `manual_review` for low-confidence, sensitive, partner, high-value, or unclear leads.

## Human-Review Gates

- public copy
- claims
- pricing
- SOW/proposal
- Stripe link selection
- optional 10% alignment layer
- any expansion beyond V1

## Client Clarity OS Connection

Theo uses the generic OS loop:

```text
Client record in Airtable
-> Intake / source material
-> Blueprint extraction
-> Human review
-> Proposal / SOW
-> Stripe payment
-> Build tasks
-> Front door + intake + CRM
-> Launch / optimization
```
